## 一、项目概述

tOS Quality Workbench 是面向 tOS 系统测试团队的一站式质量管理平台，集成 Jira 数据同步、多版本多阶段管理、智能风险分析、AI 辅助决策等功能。本次迭代重点完成了 AI 能力接入、多人协同支持 和 UI 体验优化。

---

## 二、本次迭代成果

### 2.1 AI 智能分析接入（核心进展）

**目标**：接入公司 TranAI 大模型，实现质量报告自动生成。

**完成内容**：
- 对接公司 TranAI 代理（hk-intra-paas.transsion.com/tranai-proxy/v1），兼容 OpenAI API 格式
- 三项 AI 功能上线：
  - **AI 质量总结**：自动分析 Jira 数据生成结构化质量报告
  - **AI 风险解读**：深度风险分析 + P0/P1/P2 优先级建议 + 准出判断
  - **AI 周报生成**：自动生成本周质量周报
- API Key 加密存储，前端脱敏显示，后端代理转发，安全不暴露
- 支持模型：gpt-4o（默认）、gpt-5.2-chat、deepseek-v3.1、gemini-2.5-pro

**解决的技术问题**：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| API 404 错误 | 默认地址指向旧服务 | 迁移到公司 TranAI 代理地址 |
| 中文编码崩溃 | HTTP 头 latin-1 编码不支持中文 | 姓名/部门 URL 编码 |
| Internal Server Error | 异常未捕获导致 FastAPI 500 | 后端全链路异常捕获 + 前端响应容错 |
| 数据库锁冲突 | AI 长耗时请求阻塞其他查询 | SQLite WAL 模式 + busy_timeout |

### 2.2 多人局域网协同（新能力）

**目标**：团队成员可通过局域网访问同一份数据。

**完成内容**：
- Vite 开发服务器绑定 0.0.0.0，支持局域网访问
- 配置 API 代理，前端统一通过 Vite 端口转发 API 请求
- 启动脚本自动检测并显示本机局域网 IP
- 团队成员访问 http://<主机IP>:5173 即可查看相同数据

### 2.3 UI 优化

| 改进项 | 改动前 | 改动后 |
|--------|--------|--------|
| 侧边导航 | 纯图标 + 悬停 tooltip | 图标 + 文字标签，一目了然 |
| 基础信息/关键资源 | 上下两个独立区块 | 左右并排双栏，节省纵向空间 |
| 测试动作模块 | 硬编码建议列表 | 已移除（建议由 AI 动态生成） |
| SR 风险 | 无 | 新增占位模块，待接入 SRM |

### 2.4 稳定性加固

- SQLite 启用 WAL 日志模式，解决并发读写锁冲突
- 启动脚本去除中文字符，修复 Windows GBK 编码导致的启动失败
- 数据库连接超时从 30s 增至 60s，busy_timeout 5s
